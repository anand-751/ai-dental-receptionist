🎯 FULL CONVERSATION FLOW TESTED

1. User clicks "Start Call with AI"
   ↓
2. Frontend requests /conversation/start API
   ↓
3. Backend creates session with unique ID
   ↓
4. Frontend establishes WebSocket connection
   ↓
5. AI greeting displayed
   ↓
6. User types message
   ↓
7. Message sent via WebSocket
   ↓
8. Backend processes:
   • Classifies intent (booking/pricing/enquiry/reschedule)
   • Retrieves clinic info from FAISS if needed
   • Generates response with Groq LLM
   ↓
9. AI response displayed
   ↓
10. Repeat steps 6-9
    ↓
11. User clicks "End Call"
    ↓
12. Session terminated, WebSocket closed
